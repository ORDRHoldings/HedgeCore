Report Studio Redesign: Architecture Approach

The current 1,900-line monolith needs to become a modular, theme-aligned system. Which structure?

A

Tabbed Shell (Settings Pattern)

Single /reports page with 5 tabs via ?tab= URL params. Same pattern as Settings and Market Intelligence.

Tabs:
STUDIO (bind run + compose sections + live preview) → LIBRARY (30 presets, search/filter) → SAVED (version history, re-export) → SCHEDULED (delivery rules) → REGULATORY (EMIR/MiFID/Dodd-Frank)
Pros: Consistent with app patterns, URL-addressable tabs, simple mental model
Cons: Studio tab still complex (bind + compose + preview in one view)
B

Guided Pipeline (Hedge Desk Pattern)

Multi-step wizard for report creation: BIND → COMPOSE → PREVIEW → EXPORT. Like Hedge Desk's 7-phase pipeline.

Steps:
1. BIND (select run + policy + market snapshot) → 2. COMPOSE (pick preset or build custom sections) → 3. PREVIEW (live rendered report with charts) → 4. EXPORT (format picker + delivery)
Pros: Forces correct workflow order, each step is focused and simple
Cons: Extra clicks for power users who just want to re-export a saved report
C

Hybrid: Tabs + Pipeline Studio (Recommended)

Tabbed shell for navigation, but the STUDIO tab itself is a guided pipeline. Best of both worlds.

Structure:
TAB: Studio
4-step pipeline inside:
Bind → Compose → Preview → Export
TAB: Library
30 presets, search, filter
Click preset → jumps to Studio
TAB: Saved
Version history, re-export
Click report → jumps to Preview
TAB: Regulatory
EMIR, MiFID, Dodd-Frank
One-click XML/text downloads
Pros: Guided creation, quick access to saved/regulatory, consistent app patterns
Cons: Slightly more complex routing (tab + step state)

Theme Alignment (applies to all approaches)

Current: White palette (#FAFAFA page, #FFFFFF panels) with hardcoded hex values
Target: Dark theme using T tokens from tokens.ts — var(--bg-deep), var(--bg-panel), var(--text-primary), etc.
Charts: EChartsWrapper already dark — will finally match the page background
