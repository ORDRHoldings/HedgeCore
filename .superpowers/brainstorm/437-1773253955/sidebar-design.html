Sidebar: Open by Default, Readable, Visible

The sidebar should feel like a permanent instrument panel — not something you have to discover.

Current sidebar issues to fix:

  • Default closed — new users see a mystery icon rail
  • Custom SVG icons — inconsistent sizes (14px/18px mix), no standard library
  • 10 sections — too many for scan-ability
  • Sub-items only visible expanded — collapsed mode hides all context
  • Flyout panels on hover — fragile interaction, disappear when cursor moves

Icon Style

Which icon style fits an institutional terminal product?

A

Lucide Icons (current frontend standard)

Clean 24px stroke icons. Already in the project's dependency. Used by Linear, Vercel, Notion. 1400+ icons available.

import { LayoutDashboard, Shield, BarChart3, Microscope, Globe, Settings } from "lucide-react"

Already installed · Consistent 24px grid · Stroke-based, clean
B

Phosphor Icons

6 weights (thin → fill). Slightly warmer/friendlier than Lucide. Used by Framer, Cal.com. Would need new dependency.

import { ChartBar, Shield, Microscope, GearSix } from "@phosphor-icons/react"

New dependency · 6 weight options · Slightly warmer
C

Keep Custom SVGs (current)

Hand-crafted inline SVGs in AppSidebar.tsx. Unique to ORDR but inconsistent (14px/18px mix, different stroke widths).

Already there · Inconsistent sizes · Hard to maintain
