Dashboard: "What do you want to do today?"

The dashboard greets the user and guides them into the right workflow — not a wall of widgets.

A

Command Center

Top section: greeting + 3 large action cards ("New Hedge", "Monitor Hedges", "Market Overview"). Below: live KPI strip + recent activity feed. Think Bloomberg Terminal meets Notion homepage.

Pros

  • Clear guided entry — user always knows what to do
  • Action cards are large, unmissable
  • KPIs visible without scrolling

Cons

  • Less data density than current widget grid
  • Power users may want to skip the cards
B

Mission Control

Full-width greeting bar with user name + org. Below: 3 "mission cards" with live stats INSIDE the cards (e.g., "New Hedge" card shows "4 positions ready"). Bottom: compact KPI row. Everything is a doorway.

Pros

  • Cards pull you in with real data — not just labels
  • Each card previews what's waiting
  • Feels intelligent — the app knows your state

Cons

  • Requires API calls to populate card stats
  • More complex to build
C

Adaptive Briefing

The dashboard changes based on what needs attention. If hedges are drifting → "Monitor" card pulses. If positions are unhedged → "New Hedge" card highlights with count. If nothing urgent → calm overview with market summary. Like a daily briefing that prioritizes for you.

Pros

  • Feels like the app is working for you
  • Reduces decision fatigue — shows what matters NOW
  • Most "final product" feel of all options

Cons

  • Most complex to build — needs priority logic
  • Risk of being wrong about what's urgent

All options share these elements:

  • Greeting with user name + role + org
  • 3 primary actions: New Hedge, Monitor, Market
  • Live KPI strip (exposure, coverage, pending approvals)
  • Recent activity / audit feed
  • Quick access to Audit Lab (the prospect magnet)
